Creates a box menu like the one you see on the right (or bottom). The BoxMenu tag doesn't return HTML in the normal way, instead Javascript is used to generate the BoxMenu. BoxMenuId - use this if you want control of the BoxMenu's ID cssFile [boxMenu] icon - an icon that is displayed to the left of the menu item title expandIcon - an icon that is displayed to the left of the menu item title when the menu item is the selected one (expanded) animate [0] - slides the boxes to their new positions when a new box is selected (doesn't seem to work as well as it should) height [100%] width [100%] content - the menu items title icon - Overrides the BoxMenu "icon" attribute expandIcon - Overrides the BoxMenu "expandIcon" attribute selected [0] - set this to true (1) if this is the menu item that should be expanded when the page loads url - if specified, the HTML of this URL will be the content of the menu item elementId - if specified, the element with this ID will become the content of the menu item body - if specified, this will be the content of the menu item content - same as "body" Either, "body" or "content" should contain the content of the menu item, or "elementId" or "url" can be used to specify the ID or URL of the element that makes up this menu item. Using "content" is recommended over "body" and "elementId" whenever possible, as it makes the HTML more easy-to-grasp. In Internet Explorer 7 Beta 2, BoxMenu is incompatible with . <o2 header> or <o2 incJavascript> and <o2 incStylesheet> must be called on the same page as <o2 BoxMenu> in order for the box menu to work correctly.
